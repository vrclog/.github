# セキュリティポリシー

> **注意**: この文書は英語版 [SECURITY.md](./SECURITY.md) の日本語訳です。正式なセキュリティポリシーは英語版を参照してください。

私たちはセキュリティを真剣に考えています。この文書では、セキュリティ脆弱性の報告方法と、プロセスで何を期待できるかを説明します。

## 重要なお知らせ

> **vrclog は非公式のコミュニティプロジェクトであり、VRChat Inc. とは提携していません。**
>
> vrclog ツールはあなたのマシン上のローカル VRChat ログファイルを読み取ります。以下にはアクセス**しません**：
> - VRChat アカウントの認証情報
> - VRChat のサーバーや API
> - 他のユーザーのアカウントやデータ
>
> VRChat アカウントのセキュリティに懸念がある場合は、[VRChat サポート](https://help.vrchat.com/)に直接お問い合わせください。

## 対象範囲

このセキュリティポリシーは以下の脆弱性を対象とします：

- vrclog ライブラリとツール（例：`vrclog-go`）
- 公式 vrclog ドキュメントとウェブサイト
- vrclog の GitHub organization インフラストラクチャ

## 想定されるセキュリティ上の懸念

vrclog の性質（ローカルログファイルの解析）を考慮すると、潜在的なセキュリティ問題には以下が含まれる可能性があります：

### データ漏洩リスク

- **ログパスの開示**: 機密性の高いディレクトリ構造を明らかにする可能性のあるファイルシステムパスの露出
- **ユーザー名/ワールド名の漏洩**: ログからのユーザー名、ワールド名、インスタンス ID の意図しない露出
- **セッション情報**: 特定のプレイセッションを識別できる情報の漏洩

### アプリケーションセキュリティ

- **パストラバーサル**: 意図されたディレクトリ外のファイルへのアクセスを許可する脆弱性
- **コマンドインジェクション**: CLI ツールにおけるユーザー入力の不適切な処理
- **ネットワーク露出**: サーバーモードで実行する場合の意図しないネットワークアクセス可能性やデータ露出

### プライバシーの懸念

- **不十分なデータサニタイゼーション**: 機密情報の適切な編集の失敗
- **意図しないデータ収集**: ユーザーの同意なくデータを収集または送信すること

## 脆弱性の報告

### してはいけないこと

- セキュリティ脆弱性について公開 GitHub Issue を作成しない
- 対処される前に脆弱性を公開しない
- エクスプロイトコードを公開しない

### すべきこと

セキュリティ脆弱性はメールでプライベートに報告してください：

**メール**: vrclog@googlegroups.com

**件名**: `[SECURITY] 問題の簡単な説明`

### 含めるべき情報

できるだけ多くの情報を提供してください：

1. **説明**: 脆弱性の明確な説明
2. **影響を受けるバージョン**: どのバージョンが影響を受けるか（わかる場合）
3. **再現手順**: 問題を再現する方法
4. **影響評価**: 攻撃者は潜在的に何ができるか？
5. **環境**: OS、Go バージョン、ツールのバージョンなど
6. **修正案**: 修正方法についてのアイデアがあれば（任意）

## 対応プロセス

### タイムライン

| フェーズ | 期間 |
|---------|------|
| 受領確認 | 7 日以内 |
| 初期評価 | 14 日以内 |
| 修正開発 | 重大度により異なる |
| 公開開示 | 修正リリース後 |

### 期待できること

1. **受領確認**: 7 日以内に報告の受領を確認します
2. **コミュニケーション**: 進捗状況をお知らせします
3. **クレジット**: あなたの許可を得て、セキュリティアドバイザリーでクレジットを記載します
4. **協調的開示**: 公開開示のタイミングについてあなたと協力します

### 重大度評価

脆弱性は以下に基づいて評価します：

- **影響**: 悪用によってどのような被害が生じる可能性があるか？
- **悪用可能性**: 悪用はどれほど容易か？
- **範囲**: 何人のユーザーが影響を受けるか？

## サポートされるバージョン

以下のバージョンにセキュリティアップデートを提供します：

| バージョン | サポート |
|-----------|---------|
| 最新リリース | はい |
| 前のマイナーバージョン | ベストエフォート |
| 古いバージョン | いいえ |

常に最新バージョンの使用を推奨します。

## ユーザー向けセキュリティベストプラクティス

vrclog ツールを使用する際：

1. **ソフトウェアを最新に保つ**: 常に最新バージョンを使用
2. **ログに注意する**: VRChat ログには個人情報（ユーザー名、ワールド名）が含まれています。編集せずに公開しないでください
3. **ネットワーク露出**: サーバーモードで実行する場合、適切なファイアウォール設定を確保
4. **設定の確認**: 意図しない設定がないか、定期的に設定ファイルを確認

## 謝辞

脆弱性を責任を持って開示するセキュリティリサーチコミュニティの努力に感謝します。有効なセキュリティ問題を報告したコントリビューターは、（許可を得て）リリースノートで謝辞を記載します。
