# ガバナンス

> **注意**: この文書は英語版 [GOVERNANCE.md](./GOVERNANCE.md) の日本語訳です。内容に相違がある場合は、英語版が正式な基準となります。

この文書は、vrclog プロジェクトがどのように運営され、意思決定がどのように行われるかを説明します。

## 概要

vrclog はコミュニティ主導のオープンソースプロジェクトです。プロジェクトの運営方法と、コントリビューターがどのようにより深く関わることができるかについて、透明性を保つことを目指しています。

## 役割

### コントリビューター

何らかの形でプロジェクトに貢献するすべての人：

- Issue の作成やディスカッションへの参加
- Pull Request の提出
- ドキュメントの改善
- 他のユーザーの支援
- 翻訳の提供

コントリビューターはプロジェクトの生命線です。どんなに小さな貢献でも感謝しています。

### メンテナー

メンテナーは、プロジェクトへの継続的な貢献を示してきたコントリビューターです。追加の責任があります：

- Pull Request のレビューとマージ
- Issue のトリアージ
- リリースの作成
- プロジェクトの方向性の指導
- 行動規範の執行

メンテナーは [CODEOWNERS](./CODEOWNERS) ファイルに記載されています。

## 意思決定

### 日常的な決定

日常的な事項（バグ修正、小さな改善、ドキュメント更新）については：

- どのメンテナーでも PR をレビューしてマージできます
- 良い判断を使用してください – 疑問がある場合は、セカンドオピニオンを求めてください

### 重要な決定

プロジェクトにより広く影響を与える変更については、ディスカッションを通じてコンセンサスを求めます：

**重要な決定の例：**

- 後方互換性を破る API の変更
- イベントスキーマやデータ構造の変更
- 新しい主要機能の追加
- プロジェクトガバナンスの変更
- ライセンスの変更

**プロセス：**

1. 提案された変更を説明する Issue を作成
2. コミュニティでの議論の時間を確保（主要な変更の場合、通常 1〜2 週間）
3. メンテナーが議論をまとめ、決定を下す
4. 進める前に Issue に決定を文書化

### 対立の解決

メンテナー間で決定について意見が分かれた場合：

1. 関連する GitHub Issue でオープンに問題を議論
2. コミュニティからの意見を求める
3. コンセンサスに達しない場合、最も関連する専門知識を持つメンテナーが最終決定を下す
4. 新しい情報が出てきた場合、決定を再検討できる

## リリースポリシー

[セマンティックバージョニング（SemVer）](https://semver.org/)に従います：

- **MAJOR** バージョン：互換性のない API の変更
- **MINOR** バージョン：後方互換性のある新機能
- **PATCH** バージョン：後方互換性のあるバグ修正

### バージョンガイドライン

- **v0.x.y**: 開発フェーズ。マイナーバージョン間で API が変更される可能性があります。
- **v1.0.0+**: 安定した API。破壊的変更には新しいメジャーバージョンが必要です。

### リリースプロセス

1. メンテナーがリリースノートを準備
2. バージョンが git でタグ付け
3. リリースが GitHub Releases に公開
4. アナウンスを投稿（該当する場合）

## メンテナーになるには

新しいメンテナーを歓迎します！正式な申請プロセスはありませんが、メンテナーは通常以下に基づいて招待されます：

- **継続的な貢献**: 時間をかけた定期的で質の高い貢献（コード、レビュー、ドキュメント、翻訳）
- **コミュニティへの関与**: 他者の支援、ディスカッションへの参加、ポジティブな存在であること
- **プロジェクトの価値観との整合**: 行動規範に従い、プロジェクトの目標を支援すること
- **技術的判断力**: コードレビューや技術的決定における良い判断を示すこと

メンテナーになることに興味がある場合、最善のアプローチは：

1. 定期的に貢献を始める
2. Issue のトリアージやコードレビューを手伝う
3. ディスカッションに積極的に参加
4. 既存のメンテナーに興味を表明

現在のメンテナーは、招待を行う前に議論してコンセンサスに達します。

## ガバナンスの変更

このガバナンス文書は、プロジェクトの発展に伴って更新される可能性があります。ガバナンスの変更は、他の重要な決定と同じプロセスに従います。
